# Smart Expense Tracker — Implementation Plan

## 1. Project Overview

**App Name:** Smart Expense Tracker
**Platform:** Flutter (Android & iOS)
**State Management:** Provider (beginner-friendly, easy to scale later to Riverpod/Bloc)
**Local DB:** sqflite
**Cloud DB:** Firebase Firestore (optional sync)
**Auth:** Firebase Auth
**Notifications:** flutter_local_notifications + Firebase Cloud Messaging

## 7. Notification Logic Flow

**Local Budget Alert:**
1. On every new transaction added → `budget_provider.dart` recalculates category/overall spend
2. If spend crosses 80% → trigger local notification via `notification_service.dart`
3. If spend crosses 100% → trigger a second, distinct local notification
4. Flags (`alert_sent_80`, `alert_sent_100`) prevent duplicate alerts in the same month

**Daily Reminder (Local Scheduled):**
- Scheduled once at app first-launch/login using `flutter_local_notifications` zonedSchedule
- Fires daily at a user-configurable time (default 8 PM): "Don't forget to log today's expenses"

**FCM (Server-Triggered):**
- Requires a minimal backend (Firebase Cloud Function) that runs on a schedule (e.g., monthly)
- Sends push notification: "Your monthly report is ready" → deep-links to Dashboard

## 9. Suggested Build Order (Step-by-Step)

1. Set up Flutter project + folder structure above
2. Configure Firebase (flutterfire configure) — Auth, Firestore, Messaging, Storage
3. Build `db_helper.dart` and test local DB with dummy data
4. Build Auth screens (login/signup/guest mode) + `auth_provider.dart`
5. Build Category module (predefined + custom categories)
6. Build Add/Edit/Delete Transaction flow with local DB
7. Build Transaction List screen with filter/search
8. Build Dashboard with fl_chart (pie + line chart)
9. Build Budget module + local notification triggers
10. Add FCM setup + test push notification via Firebase Console
11. Add Firestore sync (upload local → cloud when logged in)
12. Add Export (CSV/PDF) + Settings screen
